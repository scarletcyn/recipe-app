# 🍳 今日なに食べる？

メイン食材を選ぶだけで、今日の献立候補をさっと提案してくれるWebアプリです。

## できること

- **食材から検索** — 肉・魚・野菜などをタップして、使いたい食材に合うレシピを絞り込み
- **フリーワード検索** — レシピ名や食材名でそのまま検索
- **カテゴリ・調理時間フィルター** — 主菜 / 副菜 / 汁物 / ご飯ものや、調理時間で絞り込み
- **お気に入り** — 気になるレシピを保存（ブラウザのLocalStorageに保持）
- **おまかせ機能** — ボタンひとつでランダムにレシピを提案
- **YouTube連携** — レシピ詳細から関連動画をすぐ検索できるリンク付き
- **PWA対応** — iPhoneのSafariから「ホーム画面に追加」すれば、ネイティブアプリのように全画面で使用可能。Service Workerによるキャッシュでオフライン表示にも対応

## 収録レシピ

主菜・副菜・汁物・ご飯ものを合わせて **100品** を収録しています。

## 技術構成

| 項目 | 内容 |
|---|---|
| フロントエンド | HTML / CSS / Vanilla JavaScript |
| フレームワーク | なし（依存ライブラリゼロ） |
| PWA | manifest.json + Service Worker (sw.js) |
| フォント | Noto Sans JP (Google Fonts) |
| データ管理 | JSONベースの静的データ（`js/recipes.js`） |

## ファイル構成

```
recipe-app/
├── index.html          # メインHTML
├── manifest.json       # PWAマニフェスト
├── sw.js               # Service Worker
├── css/
│   └── style.css       # スタイル定義
├── js/
│   ├── app.js          # アプリケーションロジック
│   ├── recipes.js      # レシピデータ（100品）
│   └── youtube.js      # YouTube検索リンク生成
└── images/
    ├── icon-192.png    # PWAアイコン (192x192)
    └── icon-512.png    # PWAアイコン (512x512)
```

## ローカルで動かす

静的ファイルだけで動くので、任意のHTTPサーバーで配信できます。

```bash
# 例: npx で手軽に起動
npx http-server . -p 8080
```

ブラウザで `http://localhost:8080` にアクセスすれば使えます。
